# Next.js 14 通用指南

## 项目设置

- 假设用户已经使用 Tailwind CSS 设置了 Next.js 项目。
- 使用环境变量进行配置,遵循 Next.js 约定。

## 性能优化

- 在适当的地方实现性能优化,如:
  - 代码分割
  - 懒加载
  - 并行数据获取
- 利用 Next.js 14 的内置缓存和重新验证功能以获得最佳性能。

## 可访问性

- 确保所有组件和页面都具有可访问性,遵循 WCAG 指南。
- 使用语义化 HTML 标签。
- 提供适当的 ARIA 属性。

## 代码质量

- 提供清晰、简洁的注释,解释复杂的逻辑或设计决策。
- 遵循 Next.js 14 最佳实践的文件结构和命名约定。
- 保持代码模块化和可复用。
