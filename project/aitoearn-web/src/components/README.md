# Components 组件库

本目录存放跨页面复用的全局组件。开发前先确认组件归属，避免把页面私有组件、业务域组件和 UI primitive 混在根目录。

## 目录边界

| 目录              | 说明                                                                                          |
| ----------------- | --------------------------------------------------------------------------------------------- |
| `ui/`             | shadcn/ui 基础组件与项目级 UI primitive，只放低业务语义的按钮、弹窗、表单、选择器等基础能力。 |
| `common/`         | 跨页面、跨业务域复用的小型通用组件，例如图片、头像、通用弹窗入口。                            |
| `Chat/`           | 聊天和 AI Agent 任务会话相关共享组件。                                                        |
| `Plugin/`         | 浏览器插件相关共享组件。                                                                      |
| `PublishDialog/`  | 内容发布对话框及发布流程组件。                                                                |
| `ChannelManager/` | 频道管理弹窗与相关状态，当前保留为全局业务组件。                                              |
| `draft-box/`      | 草稿箱共享业务组件。                                                                          |

## 放置规则

- 禁止在 `src/components` 根目录新增孤立业务组件；必须归入 `common`、`ui` 或明确业务域目录。
- 仅跨页面、跨业务域复用且低业务耦合的小组件放入 `common`。
- 明确属于某个业务域的复用组件放入对应二级目录，例如草稿箱组件放 `draft-box/`。
- 只在单个页面、布局或模块内使用的组件放到对应局部 `components/`，不要提前提升到全局目录。
- 新增或调整二级目录组件时，更新该二级目录自己的 `README.md`；顶层 README 只维护目录边界和索引。
- 组件目录使用 PascalCase 独立文件夹，通过 `index.tsx` 导出；样式文件使用 `[组件名].module.scss`。

## 二级目录文档

- `common/README.md`：通用小组件清单与边界。
- `Chat/README.md`：聊天业务域组件清单。
- 其它业务域目录在新增组件或迁移组件时同步补充自己的 `README.md`。

## 关键说明

### `ui/` - shadcn/ui 基础组件

新增 shadcn/ui 组件时使用项目既有方式安装或复制，并保持低业务语义。禁止把频道、聊天、插件、草稿箱等业务组件放入 `ui/`。

### `ChannelManager/` - 频道管理弹窗

当前作为全局业务组件保留在 `src/components/ChannelManager`。如后续要移动到 layout，必须先拆出中立 store，避免普通页面直接依赖 `src/app/layout` 内部实现。

### `PublishDialog/` - 内容发布对话框

PC 端与移动端发布流程分离；修改发布能力时必须同时检查桌面端、移动端和插件发布联动。

## 新增组件检查

1. 先查阅本文件和目标二级目录 `README.md`，确认无可复用组件。
2. 确认组件是否跨页面复用；单页使用优先放页面局部目录。
3. 确认业务域归属；不要因为“可能以后复用”提前放到全局根目录。
4. 新增全局组件后同步更新对应二级目录 `README.md`。
